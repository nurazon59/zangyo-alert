# requirement

## 目的

4・5・6月の報酬実績から **社会保険の等級** を試算し、

- 残業時間がいくらまで許容されるか（等級が上がる手前のライン）
- 等級に紐づく支援系福利厚生の利用上限

を CLI から確認する。等級が意図せず上がるのを避けるためのアラート用途。

## 背景（ざっくり）

- 健康保険・厚生年金の **標準報酬月額（等級）** は、4/5/6月の報酬の平均で決まる（定時決定）
- この3ヶ月の残業代が膨らむと等級が上がり、社会保険料が年単位で増える
- 会社の支援系福利厚生（住宅支援等）には等級に応じた利用上限があり、等級が変わると枠も変わる

## 入力

### 1. config (json) — 固定値の初期値

- `BaseSalary` — 月額固定給
- `FixedOvertime` — 固定残業時間（**換算h**, V/W列と同じ次元）。例: 生40h × 1.25 = 50.0
- `FixedOverSalary` — 固定残業代
- `FixedBenefits` — 固定支給（住宅・通勤等）
- `HourlyRate` — 通常時給（必須）。割増前の基礎時給
- `HolidayDailyHours` — 休日出勤1日あたりの所定時間（I2列に乗算）
- `OvertimeRates.LateNight` — 深夜割増の **追加分** 倍率（例: 0.25）
- `OvertimeRates.Holiday` — 法定休日の倍率（例: 1.35）

### 2. 実績 (xlsx import, 任意)

勤怠/給与系の xlsx（例: `RPT_-_TT_-_勤怠実績[月集計]_本人用_今月_G_E_X_C.xlsx`）を `--import path.xlsx` で読み込み、該当月の実績値を初期値として埋める。

残業代は割増区分ごとに分解して再計算する:

- 平日: V2/W2（換算h、割増込み）の合計から `FixedOvertime` 超過分のみ請求対象
- 深夜: Q2 × `OvertimeRates.LateNight`（追加割増のみ）
- 休日: `Z2 + I2 × HolidayDailyHours` × `OvertimeRates.Holiday`
- 残業代 = (上記合計) × `HourlyRate`
- 見込み総支給 = `BaseSalary + FixedOverSalary + FixedBenefits + 残業代`

### 3. 対話入力 (4 → 5 → 6 月の順)

各月について以下を順に確認:

```
[4月] 実績で入力しますか？ (y=実績 / n=見込み)
  実績の場合: 総支給額を入力（xlsx import済みなら値が候補表示）
  見込みの場合: 追加残業時間 (h) を入力 → 固定値から自動計算
[5月] ...
[6月] ...
```

入力済みの値は確認のうえ次へ進む。Enterで候補値を採用。

### 4. 福利厚生の対話入力

福利厚生は **「1回利用するごとに一定額が給与に加算される」** 想定。
利用回数次第で報酬月額が増え、等級判定に影響する。

4/5/6 各月の入力時、または月入力後にまとめて、config 定義の項目ごとに利用回数を聞く:

```
[4月 福利厚生]
  住宅支援 (¥10,000/回) を何回利用？ → 2     → +¥20,000
  食事補助 (¥500/回)    を何回利用？ → 10    → +¥5,000
[5月 福利厚生] ...
```

各月の総支給額に上記加算分を足したうえで 4/5/6 平均 → 等級判定する。

**項目定義は config 側で完結**（住宅支援・食事補助 等はハードコードしない）。

config 例:

```json
"Benefits": [
  { "Name": "住宅支援", "UnitAmount": 10000 },
  { "Name": "食事補助", "UnitAmount": 500 }
]
```

- `Name` — CLIで表示される項目名
- `UnitAmount` — 1回利用あたり給与に加算される額

出力では「あと何回使うと等級が上がるか」をアラートする。

## 出力（ざっくり）

```
[算定]  4月: 実績 ¥XXX,XXX  5月: 見込 ¥XXX,XXX  6月: 見込 ¥XXX,XXX
[平均]  ¥XXX,XXX
[等級]  健保 N級 / 厚年 M級   標準報酬月額 ¥XXX,000

[アラート]
  - 次の等級まであと ¥X,XXX
  - 残り月で許容できる追加残業: 約 N 時間（時給 ¥YYY換算）
  - 住宅支援: あと N 回まで利用可
  - 食事補助: あと N 回まで利用可
```

## スコープ

- 東京都・協会けんぽの料率表（`data/social-insurance-tokyo-2026.csv`）を等級表として使用
- 介護保険料・賞与・遡及支給・産育休等の特殊ケースは **対象外**
- 福利厚生の等級⇔上限のマッピングは config で持つ（会社固有のため）

## 非スコープ

- 給与計算そのもの
- 年末調整・所得税
- 複数年の試算

## 使い方（想定）

```
zangyo-alert --config ./config.json
zangyo-alert --config ./config.json --import ./勤怠実績.xlsx
```

起動後、4 → 5 → 6 月の順に対話で実績/見込みを入力 → 結果出力。
